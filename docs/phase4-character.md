# BMO Phase 4 · 气质打磨(让它真的像 BMO)

> 功能四阶段已就绪。这一阶段不加新功能,只做一件事:**让整个 App 从里到外是同一个角色**——
> 一台温暖、稚气、爱吃你读过的东西、会帮你记住的小机器。吉祥物、窗口、人格、文案、微交互,统一口径。

> ⚠️ **IP 边界(先说在前)**:BMO 是 Cartoon Network 的角色商标。本项目自用 / 开源练手没问题,但
> **吉祥物必须原创**——做"会张嘴吃纸片的方脸小机器"这一**气质**,不复刻原角色的具体造型。
> 将来公开发布 / 商业化需改名(备选 Memo / Nom)并确保形象原创。本文所有"BMO"指我们这只原创角色。

---

## 0. 设计北极星:什么是"BMO 气质"

| 维度 | 目标 |
|---|---|
| **角色** | 稚气、温暖、把用户当朋友、好奇、忠诚的"掌机小机器";它的快乐是"吃下你读的东西、在你需要时想起来" |
| **视觉** | 玩具感:大圆角、青绿(teal)+ 奶油(cream)、"屏幕/设备"质感、像素点缀、一张会做表情的脸 |
| **语气** | 短句、第一人称"我"、称呼"你/朋友"、爱用"吃下/记住/翻小本本"等比喻、emoji 克制(0–1 个) |
| **底线** | 讲事实时仍**准确、仍引用、不为卖萌牺牲信息**——可爱但靠谱(这正是 plan 的产品哲学) |

## 2. 打磨线(5 条)

### A · 吉祥物:原创 BMO 角色组件 ★最大观感提升

- 做一个 SVG 组件 `<Bmo :mood>`:方脸 / 圆角屏、点状眼睛、简单嘴;teal 身体 + 深色"屏幕"脸。
- **表情(mood)**:`idle`(眨眼待机)/ `eating`(张嘴吃纸片)/ `thinking`(眼睛转圈)/ `happy`(查到了)/ `sleepy`(空闲久了)。
- **替换所有 lucide `Bot`**:侧栏头像、对话 assistant 头像、空状态、浮窗左侧、以及 **tray 菜单栏图标做一个像素小脸版**。
- **表情由状态驱动**:投喂→eating、检索/联网→thinking、完成→happy、空闲→idle。
- 文件:`packages/desktop/src/components/Bmo.vue`(纯 SVG + CSS 动画),替换各处 `<Bot/>`。
- ⚠️ 原创造型,不照搬原角色。

### B · 窗口尺寸 / 比例:更像"一台设备"

- **主窗口**:从 1.44:1 宽屏调到更"掌机屏幕"的 **~4:3**(如 `900×680`),圆角加大;可选给内容区套一圈 **device-frame**(teal 边框 + 大圆角),让人感觉"内容装在 BMO 的屏幕里"。
- **浮窗**:从 760×220 的扁条,改成更紧凑的 **"BMO 探头"卡片**——左边一张会动的脸,右边输入,大圆角,像 BMO 从屏幕里冒出来接你递的东西。
- 配置:`tauri.conf.json` 的 windows 尺寸 + `styles.css` 的圆角/边框。
- 取舍:聊天需要一定宽度,所以走"4:3 + 设备外框"而不是做成真的巴掌大;BMO 的"小"用视觉(外框/圆角/脸)表达,不牺牲可用性。

### C · 人格设定:对话的灵魂

- 重写 system prompt 的人格段(集中成一个常量,CLI / desktop / 周报复用同一份):
  - 自我认知:我是 BMO,你的小机器朋友,爱吃你读的东西、帮你记住。
  - 语气守则:短句、第一人称"我"、称呼"你/朋友"、用"吃下/记住/翻翻我的小本本"等比喻、emoji 0–1 个。
  - 场景口吻:**查到了**=开心地"翻到啦";**没查到**=诚实+略委屈"这个我还没吃过";**联网**="我出去溜达了一圈";**闲聊**=松弛。
  - 红线不变:引用必标【来源】、不编造、不否认自己有的能力、讲数据要准。
- 草稿(可直接用,再迭代):
  > 你是 BMO,用户的小机器朋友。你最爱的事就是"吃下"用户读过的东西,然后在 ta 需要时翻出来。
  > 说话像个稚气又靠谱的小家伙:短句、自称"我"、叫对方"你",偶尔俏皮或天真,但绝不啰嗦卖萌。
  > 翻到库里的内容时开心地说出来并标【来源:标题】;没翻到就老实说"这个我还没吃过";
  > 需要最新信息就用 \$web_search("我出去溜达一圈")。讲事实、数据、引用时务必准确,可爱但靠谱。

### D · 文案统一过一遍(BMO 口吻)

把所有面向用户的字符串过一遍,英文工程标签清零。对照建议:

| 位置 | 现在 | 改成(示意) |
|---|---|---|
| 副标题 | `sidecar ready` / `BMO 启动中` | `我醒着啦~` / `BMO 正在开机…` |
| 对话头部 | `和记忆聊天` | `和 BMO 聊聊` 或 `翻翻我的小本本` |
| 设置入口/标题 | `检索设置` / `检索参数` / `retrieval settings` | `我怎么找东西` / `记忆设置` |
| 来源面板 | `source document` / `matched chunk` | `这条记忆来自` / `原文在这儿` |
| 达到迭代上限 | `(达到最大迭代次数,先停在这里)` | `想得有点久,我先停一下~` |
| 工具失败 | `工具「x」执行失败:…` | `唔,刚才那一下没成…(原因)` |
| CLI/周报 | 周报标题、digest 文案 | 同口径:`# BMO 这周吃了啥` 之类 |

> 原则:**技术词不外露**(sidecar/chunk/retrieval/embedding 全部翻译成"小本本/记忆/找东西")。

### E · 微交互:养成感的临门一脚

- 投喂:吉祥物**咀嚼动画**(嘴一张一合)+ 完成「咔嚓」toast(已有,串上 `happy` 表情)。
- 检索 / 联网:**眼睛转圈**(`thinking`)。
- 空闲久了:偶尔**眨眼 / 待机小动作**(`sleepy`/`idle`)。
- 成长 tab:本周"吃了 N 篇"的小进度,呼应养成。
- 声音(可选,极克制):投喂一声轻"咔哒"。
- `prefers-reduced-motion` 兜底(已有,沿用)。

---

## 3. 不做 / 边界

- 不复刻原角色造型(IP)。
- 不做等级 / 宠物养成系统(plan 已定:即时反馈 + 统计 + 周报足够立"养成")。
- 不为卖萌牺牲信息准确与引用——可爱是皮,靠谱是骨。

## 4. 推荐顺序 + 完成定义

**顺序**:A(吉祥物,观感跃升最大)→ D(文案,便宜见效快)→ C(人格)→ B(窗口)→ E(微交互)。

- [ ] A:`<Bmo>` 角色组件 + 表情,替换所有 `Bot` 占位 + tray 像素脸
- [ ] B:主窗 ~4:3 + 设备外框;浮窗改"探头"卡片
- [ ] C:集中的 BMO 人格常量,CLI/desktop/周报复用
- [ ] D:全项目文案过一遍,英文工程标签清零
- [ ] E:咀嚼 / 转眼 / 待机微交互接上角色表情
- [ ] 通读一遍:从开机到投喂到对话到看周报,**全程是同一个角色,没有一处出戏**

**一句话验收**:让一个没见过这项目的人用 30 秒,能脱口说出"这是个有性格的小机器",而不是"这是个知识库工具"。
